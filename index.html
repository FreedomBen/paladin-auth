Secure Vault Primitives

Paladin Auth supports two secure vault storage architectures. Change settings instantly at any point without reformatting.

📂

Plaintext Storage Mode

Stored in a plain binary bincode file. Highly protected on Linux filesystem via strict 0600 (owner-only write/read) permissions and an automated 0700 parent directory lock.

Run in your Terminal

Switch interfaces gracefully. Control your OTP pipeline with command flags or step into a full-interactive crossterm shell.

bash — user@paladin-auth-node:~
$paladin-auth init

Cryptographic & Security Rigor

Meticulously designed primitives. Standardized vectors verified across unit tests and sandbox rules.

XChaCha20-Poly1305

Encrypted vaults are wrapped using XChaCha20-Poly1305 authenticated encryption. Master keys are stretched utilizing high-memory Argon2id password-hashing configurations. Zero exposure of metadata.

Memory Hygiene

All sensitive credentials and raw encryption keys route through Zeroize/secrecy modules, wiping memory immediately after use. Secret data types strictly enforce no Serialize and no leaky Debug structures.

Completely Sandbox Locked

Paladin Auth uses an explicit compilation deny layer checking all code dependencies. The application compilation breaks instantly if any imported module attempts to open standard sockets or load any network stacks.

Atomic File Transactions

Disk operations are isolated. Vault writes compile account data, write dynamically to transient files, verify hashes, and swap file locations atomically. In the event of crash or power loss, a .bak recovery persists.

Standard RFC Conformant

Full verification of standard TOTP (RFC 6238) and HOTP (RFC 4226) hashes using standard test matrices. Supports Aegis plaintext imports, Google Authenticator QR scans, and standardized export streams.

User Hardening

Advanced security primitives (such as automated vault locking on terminal timeouts and clipboard clearing buffers) are strictly opt-in, protecting stateless environments while preserving full configurability.
